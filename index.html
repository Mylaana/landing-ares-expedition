Une web app pour jouer à Terraforming Mars - Ares Expedition, jusqu'à 6 joueurs en ligne.

Voir le code sur GitHub

Bienvenue sur Mars

Pourquoi ce projet

Passionné de nouvelles technologies et suite à ma reconversion en développeur backend, je me suis lancé le défi de développer une application de A à Z.

L'Objectif: Apprendre un langage/framework front-end de zéro, pour répondre aux besoins du marché et compléter mes connaissances. Etant déja geek à la base, pourquoi ne pas développer un jeu ?!

C'est ainsi que mi 2024 je suis monté dans la fusée vers mars, et comme apprendre Angular en autodidacte n'était visiblement pas encore assez compliqué, pourquoi ne pas ajouter Java, SpringBoot, deux APIs, du temps réel, du DevOps, des tests automatisés, gérer tous mes visuels en SCSS, optimiser la conso de bande passante, la scalabilité, la sécurité etc...

Vous l'aurez compris, si c'est compliqué, c'est encore trop simple !

Fonctionnalités principales

Stack moderne

Frontend Angular 17.

Backend Java 23, SpringBoot.

Api REST + Websocket.

En temps réel et asynchrone

Connexions multijoueurs et communication client-serveur via Stomp Websocket.

Synchronisation d'état

Gestion de l’état de jeu de tous les joueurs par le serveur centralisé.

Architecture et Design patterns

SOLID, MVC, architecture modulaire.

Event Driven Architecture.

DevOps

CI/CD via Github Actions et AWS Cloud. Tests automatisés du code via pipeline.

Optimisation et scalabilité

Optimisation des flux de donnés et calculs côté serveur.

Visuels générés à 99% en CSS/SCSS (très peu d'images).

Aperçu du gameplay

Stack technique

  • Frontend: Angular 17, Typescript, RXJS, RX-Stomp
  • Backend: Java 23, Spring Boot, Websocket Stomp, API REST
  • DevOps: Github Actions, AWS, Pipeline CI/CD
  • Tests automatisés: Jasmine + Karma, JUnit
